# 데이터모델링

---

## 정의
'현실 세계'를 단순화하여 표현하는 기법

## 특징
- <span class="red">추상화</span>
  <span class="small" style="display: block;">일정한 형식, 간략하게 표현</span>
- <span class="red">단순화</span>
  <span class="small" style="display: block;">정해진 표기법, 단순 & 쉽게 표현, 불필요 제거</span>
- <span class="red">명확화</span>
  <span class="small" style="display: block;">애매모호함 제거, 정확하게 현상 기술</span>

## 목적
- 업무 설명, 분석, 형상화 목적
- 개발 및 데이터관리에도 사용

## 유의점
- 중복(Duplication)
  
  동일 데이터 엔티티가 중복 저장되면 안된다

- 비유연성(Inflexibility)

  사소한 변경에도 수시로 변경되면 안된다

- 비일관성(Inconsistency)
  
  연관 관계대해 명확하게 정의

## 관점
- 데이터 관점
  
  어떤 데이터들 업무 얽혀있는지

- 프로세스 관점
  
  실제 처리하고 있는 일이 무엇인지

- 데이터 & 프로세스 상관점
  
  프로세스 흐름따라 데이터가 어떤 영향이 받는지

## 중요 요소
- Things : 대상 (Entity)
- Attribute : 속성
- RelationShips : 관계

## 모델링 3가지 단계
1. <span class="blue bu">개념적</span> 데이터 모델링
   - 전사적 수행, 추상화 레벨 가장 높음

2. <span class="blue bu">논리적</span> 데이터 모델링
   - 키, 속성, 관계 표현 단계 -> <span class="blue-text">정규화</span> 활동 이뤄지는 단계
   - 논리 데이터 모델 대상으로 정규화

3. <span class="blue bu">물리적</span> 데이터 모델링
   - 실제 DB 구현, 성능, 가용성 등 물리적 요소 고려

# 데이터 스키마 단계 따른 독립성

## 스키마란?
- 기본적인 테이블 구조를 정의한 것

## 데이터 스키마 구조

### USER
- <span class="bu">외부 스키마</span> : <span class="bu">각(여러) 사용자</span>가 보는 스키마 정의 및 표현
- <span class="bu">개념 스키마</span> : <span class="bu">모든(NOT 여러) 사용자</span>가 보는 데이터 정의, 표현, 관계 정의
- <span class="bu">내부 스키마</span> : 물리적 저장 구조 단계 -> <span class="bu">저장구조, 컬럼, 인덱스 정의</span>

### DB
- <span class="bu red">논리적</span> 독립성
  - 개념 스키마 변경 -> 외부 스키마 영향 X
  - 외부 <-> 개념
- <span class="bu red">물리적</span> 독립성
  - 내부 스키마 변경 -> 개념/외부 스키마 영향 X
  - 내부 <-> 개념, 외부

# 엔티티

## 정의
- 업무 사용 데이터들을 용도별로 분류한 그룹

## 특징
- 업무에서 쓰이는 정보
- 식별자 존재
- 2개 이상 인스턴스
- 반드시 속성 필요
  - 이때 하나 인스턴스는 2개 이상 속성 필요
    - 즉, 하나 엔티티 2개 이상 속성 필요
- 다른 엔티티와 1개 이상 관계

## 분류 방법
- 유형, 무형에 따른 분류 > 개사유 (계셔유)
  - **개념** 엔티티 : 모델링 대상 **형태 없음**
    - 부서, 학과
  - **사건** 엔티티 : 모델링 대상 **행위로 인해 발생**
    - 주문, 이벤트 응모
  - **유형** 엔티티 : 모델링 대상 **물리적인 형태 존재**
    - 상품, 회원
- 발생 시점에 따른 분류 > 행기중
  - **행위** 엔티티 : <span class="bu red">2개 이상</span> 엔티티로 부터 파생
    - 주문 내역, 이벤트 응모 이력
  - **기본** 엔티티 : 모델링 대상이 <span class="bu red">원래 존재하는 요소</span>
    - 상품, 회원, 부서
  - **중심** 엔티티 : 모델링 대상의 <span class="bu">업무 과정 중 하나</span>, **기본** 엔티티로부터 <span class="red">파생</span>, **행위** 엔티티 <span class="red">생성</span>
    - 주문, 매출, 예약

## 엔티티 명명 주의점
- **업무 실제 사용 용어**
- <span class="bu">한글 약어 사용 X</span>, 영어 대문자 표시
- <span class="bu">단수 명사</span> 표현
- 띄어쓰기 X
- 의미상 중복 X (주문, 결제는 가능)
- 명확 표현

# 속성

## 정의
- 엔티티 특징 표현 최소 데이터 단위

## 특징
- 더 이상 나눌 수 없는 레벨
- 업무 필요 항목
- 엔티티와 인스턴스를 설명
- 하나 속성 = 하나 속성 값
  - 여러개 가지면 **1차 정규화**
- 일반 속성은 정해진 주식별자에 함수적 종속성을 가져야한다
  - 완전 함수적 종속이 아닌 부분 종속이면 2차 정규화

## 특성에 따른 분류

### 일반적인 특성
- 기본 속성
  - **바로 정의 가능한 속성**

- 설계 속성
  - <span class="bu">인스턴스에 유니크함을 부여</span>하는 속성 (PK 토대)
  - 업무 없으나, 고유함 보전위해 필요해서 만들어짐
  - 학번, 사번 등등
